◇ Agents

Smarter Together.
Not Harder.

Glue doesn't just run one LLM in a loop — it orchestrates specialized agents, each with the right model, tools, and focus for the job.

◆ Planned Feature

The role-based agent system described below is a planned feature. Current agent orchestration uses generic spawn_subagent and spawn_parallel_subagents tools with configurable tool access — no predefined roles.

01 · Orchestration

One Agent Leads.
Many Execute.

Automatic Delegation

Your main agent recognizes when a task benefits from specialization and spawns the right subagent — a fast Scout for exploration, a thorough Reviewer for code quality, or a powerful Planner for architecture decisions.

Fresh Context, Focused Scope

Each subagent starts with a clean context window and a targeted system prompt. No conversation history bloat, no distraction. It does one job, returns the result, and the orchestrator synthesizes everything.

Total: 3.1s (parallel)

Independent & Concurrent

Spawn multiple agents of the same type to explore different parts of your codebase simultaneously. Read-only agents have no shared state conflicts — they execute in parallel and results come back in order.

Nested Delegation

Agents can spawn sub-agents up to two levels deep. A Planner can dispatch Scouts to gather context before designing. A Fixer can spawn a Researcher to look up documentation. The orchestrator prevents infinite nesting.

Cost-Efficient

Fast models for Scouts mean exploring three modules in parallel costs less than a single turn of a powerful model. Use the right tier for the job.

05 · Configuration

Your Models.
Your Rules.

# .glue/config.yaml agents: scout: provider: anthropic model: claude-haiku-4-5 reviewer: provider: mistral model: mistral-large-latest planner: provider: anthropic model: claude-opus-4-6 builder: provider: openai model: gpt-5 researcher: provider: ollama model: llama3.2

Mix & Match Providers

Every agent type can use a different model and provider. Use Claude for planning, GPT for building, Mistral for reviewing — whatever combination works best for your workflow.

Per-Project Overrides

Global defaults live in ~/.glue/config.yaml. Project-level overrides in .glue/config.yaml. A Python project might use different agents than a Dart project.

Coming Soon: Custom Roles

Define entirely new agent roles with Markdown files — custom system prompts, tool restrictions, and model selection. Think: a security-auditor agent that only runs static analysis, or a docs-writer that generates API documentation.
