Context Window Management

A three-tier automatic system that fits conversations into the LLM's context budget before each call. The original conversation is never mutated — only the view sent to the provider is affected.

Pipeline

Three-Tier Reduction Strategy

Before every LLM call, ContextManager.prepareForLlm() applies up to three tiers in order of increasing aggressiveness and cost.

Input
_conversation (full history)
Never mutated. All turns since the session started.
prepareForLlm()
Tier 1 · Always
Tool Result Trimmer
Replaces tool results >200 tokens from turns older than toolResultTrimAfter (default 3) with compact placeholders. Zero LLM cost.
estimated > 80% of inputBudget?
Tier 2 · When > 80%
Conversation Compactor
Calls a small model to summarize all but the last keepRecentTurns (default 4) user turns. Inserts a synthetic summary message.
still > 95% after compaction?
Tier 3 · When > 95%
Sliding Window Trimmer
Drops oldest complete turns until the estimate drops back to the 80% target. Prepends a "[N messages removed]" marker. Zero LLM cost.
Output
Trimmed view sent to LLM
A new list is returned. The original _conversation is untouched.

Configuration

context: settings in glue.yaml

All thresholds are user-configurable in ~/.glue/config.yaml. Source: cli/lib/src/context/context_config.dart

YAML

# ~/.glue/config.yaml context: auto_compact: true # enable Tier 2 compact_threshold: 0.80 # fire Tier 2 at 80% of inputBudget critical_threshold: 0.95 # fire Tier 3 at 95% keep_recent_turns: 4 # always keep last 4 user turns verbatim tool_result_trim_after: 3 # trim results older than 3 turns

Options

KeyDefaultEffect
auto_compacttrueEnable Tier 2 summarization. Set to false to use Tier 1+3 only.
compact_threshold0.80Fraction of inputBudget that triggers Tier 2.
critical_threshold0.95Fraction that triggers Tier 3 after Tier 2 is insufficient.
keep_recent_turns4User turns always kept verbatim, never compacted.
tool_result_trim_after3Tool results older than N turns get truncated by Tier 1.
